## 二、功能清单（按能力域）

### 1. 工程与分发

- ✅ Maven 多模块工程：`archimedes-parent` / `archimedes-core`（Java 8 字节码、零 servlet 依赖、单 jar 双端复用）/ `archimedes-spring-boot-2-starter`（javax、spring.factories）/ `archimedes-spring-boot-3-starter`（jakarta、AutoConfiguration.imports）
- ✅ Spring Boot 2.7.x 与 3.x 双版本支持，BOM 按模块隔离
- ✅ 引入即用：零配置自动装配，`archimedes.api.enabled=false` 可整体关闭
- ✅ 示例工程双份：`example`（SB3，端口 8080）、`example-boot2`（SB2.7，端口 8081）
- ⬜ 发布到 Maven 中央仓库（发布后 Maven / Gradle 项目均可直接引入坐标）
- ⬜ Gradle 消费方验证（示例或文档说明）
- ⬜ Spring Cloud 项目兼容性验证（网关/注册中心场景下的扫描行为）
- ⬜ WebFlux 响应式 `@RestController` 支持（当前仅 Servlet 栈）
- ⬜ 接口信息端点的权限控制（避免敏感信息泄露）

### 2. 接口契约扫描与展示

- ✅ REST 扫描：类/方法级路径合并、HTTP 方法、参数（`@RequestParam`/`@PathVariable`/`@RequestBody`/`@RequestHeader`，含 required 与来源标记）、返回类型、consumes/produces、`@Deprecated` 标记
- ✅ 排除规则：自身端点（`{base-path}/**`）与框架 `/error` 不出现在结果中
- ✅ `base-packages` 包前缀过滤
- ✅ JSON 端点：`GET {base-path}/apis`（默认 `/archimedes/apis`）
- ✅ 内置 UI：`GET {base-path}`，API 地址注入
- ⬜ WebSocket 扫描：`@ServerEndpoint` 注解方式
- ⬜ WebSocket 扫描：`WebSocketConfigurer`/`WebSocketHandlerRegistry` 注册方式
- ⬜ WebSocket 扫描：STOMP（`WebSocketMessageBrokerConfigurer`，订阅 Topic/Queue、消息类型）
- ⬜ RPC 扫描：Dubbo（ServiceConfig / MetadataService，服务名、方法签名、版本、分组、协议）
- ⬜ RPC 扫描：gRPC（Server Reflection，ServiceDescriptor）
- ⬜ TR 扫描：SOFARPC TR 协议（`@SofaService` / ProviderConfig）
- ⬜ TR 扫描：腾讯 tRPC（`@TRpcService`）
- ⬜ 统一多协议数据模型与 JSON 分组结构（restApis / webSocketApis / rpcApis / trApis）
- ⬜ 前端升级：按协议分 Tab、接口搜索与筛选
- ⬜ 前端升级：REST 在线调试
- ⬜ 前端升级：RPC 契约展示（方法签名、参数说明）

### 3. 全链路日志追踪

- ⬜ `TraceIdGenerator` SPI + 默认 UUID 实现（`@ConditionalOnMissingBean`，用户可用 Snowflake 等替换）
- ⬜ `TraceIdResolver` SPI：接入用户项目自有 traceId（从请求头/上下文解析），默认关闭、配置开启
- ⬜ Servlet Filter：请求进入生成/提取 traceId → 写入 MDC → 响应头回写 → 请求结束清理（javax 与 jakarta 两个 starter 各一份实现，逻辑下沉 core）
- ⬜ spanId 生成与 MDC 写入
- ⬜ 配置项 `archimedes.trace.*`：enabled / use-project-trace-id / header-name / response-header / mdc-key / span-id-key
- ⬜ 跨线程传递：`TaskDecorator` + `@Async` 线程池自动装配
- ⬜ 跨线程传递：`BeanPostProcessor` 自动包装容器内所有 `Executor`/`ExecutorService` 类型 Bean
- ⬜ 跨线程传递：TTL（TransmittableThreadLocal）桥接 MDC
- ⬜ 跨线程传递：`CompletableFuture`/裸线程手动包装工具类（`MdcWrappers.wrap(...)`）
- ⬜ 覆盖边界启动日志提示（`commonPool`/裸线程池不在自动覆盖范围，已决策不做 javaagent）

### 4. 日志采集与查询

- ⬜ 自定义 Logback Appender：捕获 `ILoggingEvent`（结构化，与日志格式解耦），按 MDC traceId 归集
- ⬜ `LogStore` SPI：`append` / `queryByTraceId`（分页）
- ⬜ 默认实现：内存有界环形缓冲（按 traceId 索引，容量可配置）
- ⬜ 预留实现位：Elasticsearch（`@ConditionalOnClass` 条件装配，后续接入）
- ⬜ 查询端点：`GET {base-path}/logs/trace/{traceId}`（时间排序、分页、含线程/级别/logger/spanId）
- ⬜ 辅助端点：`GET {base-path}/trace/current`（返回当前请求 traceId）
- ⬜ UI 链路追踪 Tab：traceId 查询、时间线展示、多线程日志高亮区分

### 5. 日志配置管理

- ⬜ 依赖内置默认 `logback-spring.xml`（pattern 含 traceId 占位）
- ⬜ 智能加载：`EnvironmentPostProcessor` 检测用户 classpath 是否有 `logback-spring.xml`/`logback.xml`，没有则注入内置配置（通过 `logging.config`，SB2/SB3 双注册）
- ⬜ 配置项 `archimedes.log.*`：pattern / path / max-history / max-file-size
- ⬜ 自定义 pattern 与采集解耦验证（改格式不影响按 traceId 查询）

## 四、已锁定的设计决策（约束未实现项）

| 决策 | 内容 |
|---|---|
| TR 协议范围 | SOFARPC TR 与腾讯 tRPC **两者都做**，各自独立扫描器 |
| 日志存储 | 内存环形缓冲为默认实现，`LogStore` SPI 为 Elasticsearch 预留接入位 |
| 跨线程覆盖 | 覆盖 Spring 世界（TaskDecorator + Executor Bean 包装 + TTL 桥接），**不做 javaagent**；`commonPool`/裸线程池通过工具类手动覆盖 |
| SB2 基线 | 2.7.18（多模块拆分已落地支撑） |
| 配置前缀 | 统一 `archimedes.*`（需求文档中的 `contra.*` 均对应更名） |
| 依赖形态 | 新协议扫描器一律 `<optional>true</optional>` + `@ConditionalOnClass`，宿主没引入对应框架时零影响 |
