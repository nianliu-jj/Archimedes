## 二、功能清单（按能力域）

### 1. 工程与分发

- ✅ Maven 多模块工程：`archimedes-parent` / `archimedes-core`（Java 8 字节码、零 servlet 依赖、单 jar 双端复用）/ `archimedes-spring-boot-2-starter`（javax、spring.factories）/ `archimedes-spring-boot-3-starter`（jakarta、AutoConfiguration.imports）
- ✅ Spring Boot 2.7.x 与 3.x 双版本支持，BOM 按模块隔离
- ✅ 引入即用：零配置自动装配，`archimedes.api.enabled=false` 可整体关闭
- ✅ 示例工程双份：`example`（SB3，端口 8080）、`example-boot2`（SB2.7，端口 8081）
- ⬜ 发布到 Maven 中央仓库（发布后 Maven / Gradle 项目均可直接引入坐标）
- ⬜ Gradle 消费方验证（示例或文档说明）
- ⬜ Spring Cloud 项目兼容性验证（网关/注册中心场景下的扫描行为）
- ✅ WebFlux 响应式 `@RestController` 支持（REACTIVE 条件装配分支：响应式扫描器 + 端点/UI 复用 + RPC 扫描同等生效；trace/日志采集仍为 Servlet 栈，边界已在 README 声明）
- ⬜ 接口信息端点的权限控制（避免敏感信息泄露）

### 2. 接口契约扫描与展示

- ✅ REST 扫描：类/方法级路径合并、HTTP 方法、参数（`@RequestParam`/`@PathVariable`/`@RequestBody`/`@RequestHeader`，含 required 与来源标记）、返回类型、consumes/produces、`@Deprecated` 标记
- ✅ 契约增强：请求体/响应体字段结构树（requestBodySchema/responseSchema：字段名/类型/必填/说明/集合标记/嵌套；解包 ResponseEntity/Mono/Flux/集合；深度与循环保护）；字段与参数说明零依赖反射读取 Swagger v3/v2 + Jackson + validation 注解；UI 示例 JSON 预填 + 请求/响应字段表
- ✅ 排除规则：自身端点（`{base-path}/**`）与框架 `/error` 不出现在结果中
- ✅ `base-packages` 包前缀过滤
- ✅ JSON 端点：`GET {base-path}/apis`（默认 `/archimedes/apis`）
- ✅ 内置 UI：`GET {base-path}`，API 地址注入
- ✅ WebSocket 扫描：`@ServerEndpoint` 注解方式（须为 Spring Bean，javax/jakarta 双端）
- ✅ WebSocket 扫描：`WebSocketConfigurer`/`WebSocketHandlerRegistry` 注册方式（含 SockJS 标记）
- ✅ WebSocket 扫描：STOMP（握手端点、@MessageMapping、@SubscribeMapping 目的地）
- ✅ RPC 扫描：Dubbo（ServiceBean 容器自省，覆盖注解与 XML；服务名、方法签名、版本、分组；2.7/3.x 兼容）
- ✅ RPC 扫描：gRPC（BindableService Bean 自省，方法/streaming 形态/消息类型；不依赖 Server Reflection——设计取舍见归档 change）
- ✅ TR 扫描：SOFARPC（@SofaService 反射式，interfaceType/uniqueId/bindings；零 SOFA 编译依赖）
- ✅ TR 扫描：腾讯 tRPC（@TRpcService 反射式，防御读取 name/version/group；FQCN 不匹配安全退化）
- ✅ 统一多协议数据模型与 JSON 分组结构（restApis / webSocketApis / rpcApis；RPC 四协议以 protocol 字段区分共用 rpcApis）
- ✅ 前端升级：Tab 化分区（REST / WebSocket / RPC / TR / 链路追踪）+ 全局过滤 + REST 方法筛选 + RPC/TR 协议 chip 筛选
- ✅ 前端升级：REST 在线调试（行内 try-it 面板：契约预填参数 → fetch → 状态/耗时/响应头/响应体；trace 响应头一键联动链路查询）
- ✅ 前端升级：RPC 契约展示（protocol 徽标 + 方法签名列表 + metadata 键值详情，RPC/TR 双 Tab 复用）

### 3. 全链路日志追踪

- ✅ `TraceIdGenerator` SPI + 默认 UUID 实现（`@ConditionalOnMissingBean`，用户可用 Snowflake 等替换）
- ✅ `TraceIdResolver` SPI：接入用户项目自有 traceId（可选 Bean，优先级最高）
- ✅ Servlet Filter：请求进入生成/提取 traceId → 写入 MDC → 响应头回写 → 请求结束按 key 精准清理（javax/jakarta 薄壳，逻辑下沉 core）
- ✅ spanId 生成与 MDC 写入
- ✅ 配置项 `archimedes.trace.*`：enabled / use-project-trace-id / header-name / response-header / mdc-key / span-id-key
- ✅ 跨线程传递：`TaskDecorator` + `@Async` 线程池自动装配（已有 decorator 自动组合）
- ✅ 跨线程传递：`BeanPostProcessor` 自动包装容器内所有 `Executor`/`ExecutorService` 类型 Bean（TaskScheduler 跳过，exclude-beans 逃生口）
- ✅ 跨线程传递：纯 MDC 快照传递（实现级修正：无 agent 前提下 TTL 无增量收益，零新依赖实现同等覆盖）
- ✅ 跨线程传递：`CompletableFuture`/裸线程手动包装工具类（`MdcWrappers.wrap(...)` 六种形态）
- ✅ 覆盖边界启动日志提示（@Async 退化 SimpleAsyncTaskExecutor 场景 WARN 检测；commonPool/裸池用 MdcWrappers）

### 4. 日志采集与查询

- ✅ 自定义 Logback Appender：编程式挂载 root logger，结构化捕获，与格式解耦，无 traceId 不采集
- ✅ `LogStore` SPI：`append` / `queryByTraceId`（分页）
- ✅ 默认实现：内存有界存储（双上限：全局 10000 / 单链路 500，最老链路整体淘汰，可配置）
- ✅ 预留实现位：自定义 LogStore Bean 即接入（`@ConditionalOnMissingBean` 让位，ES 客户端后续实现）
- ✅ 查询端点：`GET {base-path}/logs/trace/{traceId}`（时间升序、分页、含线程/级别/logger/spanId）
- ✅ 辅助端点：`GET {base-path}/trace/current`（返回当前请求 traceId）
- ✅ UI 链路追踪分区：traceId 查询、时间线展示、多线程按颜色区分

### 5. 日志配置管理

- ✅ 依赖内置默认配置 archimedes-logback.xml（刻意避开约定名防误拾取；pattern 含 traceId/spanId，CONSOLE+滚动 FILE）
- ✅ 智能加载：EnvironmentPostProcessor 四条让位规则（logging.config 已设/约定文件存在/非 logback/开关关闭），core spring.factories 单份注册 SB2/SB3 通用
- ✅ 配置项 `archimedes.log.*`：pattern / path / max-history / max-file-size + fallback-enabled 逃生开关
- ✅ 自定义 pattern 与采集解耦验证（真机实测改格式后查询结构化返回不受影响）

### 6. 配置展示与热更新

- ✅ 配置查询端点 `GET {base-path}/config`：按 Environment 优先级枚举全部可枚举属性源（key/value/来源），非枚举源与单键异常防御式跳过
- ✅ 敏感值脱敏：key 小写 contains 命中即 `******`（默认 password/secret/token/credential/key；`archimedes.config.sensitive-keys` 整体替换）
- ✅ 热更新端点 `POST {base-path}/config/update`：写入最高优先级动态属性源 `archimedesDynamicConfig`，value 缺省=删除覆盖恢复原值；key 空 400 / 开关关闭 403
- ✅ `@ConfigurationProperties` Bean 原地重绑定（Binder + 既有实例；构造器绑定防御式跳过 WARN）
- ✅ 事件联动：`ArchimedesConfigChangedEvent` 必发；classpath 有 Spring Cloud 时反射发布 `EnvironmentChangeEvent`（零编译依赖）
- ✅ UI Config Tab：属性源分组 + 动态覆盖高亮 + 搜索 + 行内编辑应用 + 一键移除；热更新关闭时编辑禁用
- ✅ 配置项 `archimedes.config.*`：enabled / hot-refresh-enabled / sensitive-keys
- ✅ 边界：动态覆盖不持久化（重启失效）；SERVLET/REACTIVE 双栈同等可用

### 7. 数据库监控（HikariCP + SQL，参照 Druid）

- ✅ DataSource 零侵入代理：BeanPostProcessor 自动包装全部 DataSource Bean（`archimedes.sql.exclude-beans` 逃生口；Wrapper.unwrap 全量透传保证原生实现可达）
- ✅ SQL 执行明细：归一化语句 / 绑定参数（可关）/ 耗时 / 类型（QUERY/UPDATE/BATCH/EXECUTE）/ 行数（查询 ResultSet 计数回填、更新影响行数、批处理求和）/ 异常记录后原样抛出
- ✅ traceId 关联：执行明细携带 MDC traceId，UI 一键联动链路日志查询
- ✅ 聚合统计：按「数据源+归一化 SQL」聚合次数/总耗时/平均/最大/失败数（无锁累加）；max-sql-stats 上限防膨胀
- ✅ 慢 SQL：阈值 `archimedes.sql.slow-sql-millis`（默认 1000ms）；最近执行与慢 SQL 有界环形缓冲（各 500 条，可配）
- ✅ 连接池指标：`PoolMetricsContributor` SPI + HikariCP 内置实现（optional 依赖 + @ConditionalOnClass 字符串守护；活跃/空闲/等待/总连接/池配置）
- ✅ 多数据源：按 Bean 名区分，端点与 UI 全部纳入
- ✅ 端点 `GET {base-path}/db` + UI DB Tab（池状态卡片 / SQL 统计表 / Recent/Slow 列表 / 搜索 / 手动+自动刷新）
- ✅ 配置项 `archimedes.sql.*`：enabled / slow-sql-millis / max-history-size / capture-parameters / exclude-beans / max-sql-stats
- ✅ 边界：仅 JDBC DataSource（R2DBC 不支持）；监控数据存内存重启清零；Statement 字面量 SQL 仅空白归一化

## 四、已锁定的设计决策（约束未实现项）

| 决策 | 内容 |
|---|---|
| TR 协议范围 | SOFARPC TR 与腾讯 tRPC **两者都做**，各自独立扫描器 |
| 日志存储 | 内存环形缓冲为默认实现，`LogStore` SPI 为 Elasticsearch 预留接入位 |
